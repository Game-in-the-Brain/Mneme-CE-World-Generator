# Version Logs

## Format

Each log file: `yymmdd-hhmmss.md` (e.g. `260428-225152.md`)

## When to Update

When the user says **"update the log"**, create a new log file with the current timestamp and include:

1. **Session Summary** — what was built, changed, or decided
2. **Next Actions** — prioritized list of what comes next
3. **Problems & Blockers** — anything that failed, errored, or is incomplete
4. **Root Cause Analysis (RCA)** — why each problem happened and how to prevent it
5. **Decisions Made** — architectural or design decisions with rationale
6. **Files Touched** — list of modified/created files

## Log Template

```markdown
# Version Log: yymmdd-hhmmss

## Session Summary

## Next Actions

## Problems & Blockers

## Root Cause Analysis (RCA)

## Decisions Made

## Files Touched
```

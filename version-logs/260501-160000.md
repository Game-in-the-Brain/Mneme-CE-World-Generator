Implemented **QA-079** and **QA-080** — comprehensive name generation fix for companion stars, main worlds, and circumstellar disks.

### QA-079 — Companion Star Names

**Status:** ✅ Implemented

**Problem:** Companion stars had no generated names. They displayed only as "Companion 1", "Companion 2", etc.

**Changes:**
1. **`src/types/system.ts`**: Added `companionNames?: Record<string, string>` to `PlaceNames` interface.
2. **`src/lib/placeNameGen.ts`**: `generatePlaceNames()` now generates a unique name for each companion star using `PlaceGen`, stored in `companionNames` keyed by `star.id`.
3. **`src/components/tabs/StarTab.tsx`**:
   - Displays companion star name (from `placeNames.companionNames`) alongside spectral type.
   - In edit mode, name is editable via inline text input (same pattern as system name).
4. **`src/components/SystemViewer.tsx`**:
   - Added `handleEditCompanionName` callback that updates `pendingSystem.placeNames.companionNames` in edit mode, or saves directly to the system object in view mode.
   - Passed callback to `StarTab`.
   - Legacy migration: on first view, systems without `companionNames` auto-populate with defaults ("Companion 1", etc.) into annotations.

### QA-080 — Main World and Circumstellar Disk Names

**Status:** ✅ Implemented

**Problem:** `generatePlaceNames()` skipped the main world and all circumstellar disks, leaving them with blank or placeholder names.

**Changes:**
1. **`src/lib/placeNameGen.ts`**:
   - Main world now gets a generated name stored in `bodyNames` under the synthetic ID `${system.id}-mainworld` (matching the existing ID pattern in `PlanetarySystemTab.tsx`).
   - Each circumstellar disk gets a generated name stored in `bodyNames` under `disk.id`.
2. **`src/components/SystemViewer.tsx`**:
   - Refactored the annotation auto-populate `useEffect` to be more robust:
     - Always merges from `placeNames.bodyNames` for any missing entries.
     - Explicit legacy migration for main world (fallback: system name or "Main World").
     - Explicit legacy migration for disks (fallback: "Disk N").
     - Explicit legacy migration for companion stars (fallback: "Companion N").
   - Only writes to localStorage when changes are actually made (`needsSave` flag).
